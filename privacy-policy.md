# Privacy Policy

## Project Status

Human Performance Analytics Project is currently a non-commercial development and evaluation project. The current working title of the application is Aura Elite.

This project is not currently sold, monetized, advertised, or connected to any data resale model.

## Data We May Process

The project may process the following categories of data during development and evaluation:

### Garmin-authorized data

If Garmin API access is granted in the future, the project may process user-authorized Garmin data such as:

- activity history;
- activity details;
- sleep data;
- heart rate;
- resting heart rate;
- stress;
- respiration;
- calories;
- steps;
- Body Battery when available;
- Pulse Ox when available;
- body composition when available;
- activity files such as FIT, GPX or TCX when available.

### Manually entered contextual data

The prototype may also allow users to manually enter:

- perceived fatigue;
- sleep perception;
- session RPE;
- subjective recovery;
- nutrition context;
- hydration;
- pain or discomfort notes;
- mood or stress context;
- training notes.

## Purpose of Data Use

Data are used only to support non-diagnostic wellness and performance analytics, such as:

- recovery trends;
- sleep trends;
- training load indicators;
- nutrition context;
- data quality indicators;
- longitudinal personal trends;
- contextual performance insights.

## No Medical Diagnosis

The project does not provide medical diagnosis, treatment, prescription, or clinical decision-making.

Outputs are intended as non-diagnostic trends, signals and contextual insights.

## No Sale or Resale of Data

The project does not sell, resell, rent or share user data for advertising or commercial resale.

## Garmin Passwords

The project will not request or store Garmin passwords.

If Garmin API access is granted, the integration will be based on Garmin-authorized user consent flows.

## Data Storage

During development, some sensitive manually entered data may remain local-only by default.

If cloud storage is used, it will be limited to clearly defined data categories, with appropriate access controls and user transparency.

## Sensitive Data Handling

Sensitive manually entered data, such as pain notes, subjective wellness, nutrition details, menstrual information or private notes, should be treated separately and may remain local-only by default.

## User Control

Users should be able to:

- understand what data are processed;
- export their data;
- delete their data;
- revoke access where applicable;
- request clarification about data handling.
