# Garmin API Scope

## Requested APIs

The project is requesting evaluation access to:

1. Garmin Health API
2. Garmin Activity API

## Garmin Health API

The Garmin Health API is requested for user-authorized all-day health and wellness metrics.

Potential data categories include:

- sleep;
- heart rate;
- resting heart rate;
- stress;
- respiration;
- calories;
- steps;
- Body Battery when available;
- Pulse Ox when available;
- body composition when available.

## Purpose

These data would be used to support:

- recovery trend analysis;
- sleep trend analysis;
- longitudinal wellness indicators;
- data quality scoring;
- non-diagnostic readiness and recovery insights.

## Garmin Activity API

The Garmin Activity API is requested for user-authorized activity data.

Potential data categories include:

- activity history;
- activity details;
- activity duration;
- distance;
- speed or pace;
- heart rate during activity;
- calories;
- power when available;
- cadence when available;
- activity files such as FIT, GPX or TCX when available.

## Purpose

These data would be used to support:

- training load analysis;
- activity history;
- session context;
- recovery interaction analysis;
- longitudinal performance trends.

## Data Minimization

The project intends to request only data categories needed for the analytics use case.

## User Authorization

All Garmin data access should be based on user authorization.

The project will not request or store Garmin passwords.

## Non-Diagnostic Use

Garmin data will be used for wellness, performance and training-context analytics only.

The project does not provide diagnosis, treatment, prescription or clinical decision-making.
